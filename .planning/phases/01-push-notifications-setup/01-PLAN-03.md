Step 1 — Add Node.js imports at the top of the file (after the triple-slash reference comment):
```typescript
import { readFileSync, writeFileSync } from 'fs';
import { resolve } from 'path';
```

Step 2 — Import firebaseConfig from the config module. Place this import after the existing Vite plugin imports:
```typescript
import { firebaseConfig } from './src/firebase/config';
```

Step 3 — Define the plugin function before the `defineConfig` call:
```typescript
function stampFirebaseSwPlugin() {
  return {
    name: 'stamp-firebase-sw',
    writeBundle({ dir }: { dir?: string }) {
      const outDir = dir ?? 'dist';
      const swPath = resolve(outDir, 'firebase-messaging-sw.js');
      let src = readFileSync(swPath, 'utf-8');
      src = src
        .replace("apiKey: ''", `apiKey: '${firebaseConfig.apiKey}'`)
        .replace("authDomain: ''", `authDomain: '${firebaseConfig.authDomain}'`)
        .replace("projectId: ''", `projectId: '${firebaseConfig.projectId}'`)
        .replace("storageBucket: ''", `storageBucket: '${firebaseConfig.storageBucket}'`)
        .replace("messagingSenderId: ''", `messagingSenderId: '${firebaseConfig.messagingSenderId}'`)
        .replace("appId: ''", `appId: '${firebaseConfig.appId}'`);
      writeFileSync(swPath, src, 'utf-8');
    },
  };
}
```

Step 4 — Add `stampFirebaseSwPlugin()` to the plugins array in defineConfig, AFTER VitePWA:
```typescript
plugins: [
  react(),
  VitePWA({ ... }),
  stampFirebaseSwPlugin(),
],
```

The existing VitePWA configuration (registerType, includeAssets, manifest, strategies, workbox) must remain unchanged.

IMPORTANT: `writeBundle` runs only during `vite build` — NOT during `vite dev`. In dev, the public directory is served as-is with empty strings, which is correct because vapidKey is also empty and the hook returns 'unconfigured' status, making no FCM calls.

IMPORTANT: Do NOT modify public/firebase-messaging-sw.js. The plugin stamps the dist/ copy only.
