    New (branched on error.code per D-01 + D-02):
    ```tsx
    {error && (error as { code?: string }).code === 'permission-denied' ? (
      <div className="banner warn">{t.firestoreNotEnabled}</div>
    ) : error ? (
      <div className="banner warn">{t.todosLoadError}</div>
    ) : null}
    ```

    The cast `(error as { code?: string })` is safe because:
    - `error` is `Error | null`; we already know it's truthy in the first branch
    - FirebaseError has `.code: string` at runtime; TypeScript's `Error` type does not declare it
    - Using `as { code?: string }` is the minimal cast that avoids `any` (per RESEARCH Pattern 5)
